How a static page is displayed.

When the client sends a request to the server, Django checks the URL configuration defined in urls.py. Here, a single re_path rule that uses a regex to capture every incoming URL to ex01_resolver function from the view python file: 

urlpatterns = [
    re_path(r'.*', views.ex01Resolver)
]

The view then extracts the page name directly from the URL path and builds the template filename dynamically: 

def ex01Resolver(request: HttpRequest) -> HttpResponse:
    file = f"ex01/{request.path.split('/')[2]}.html"
    if file.find("..") != -1:
        response = render(request, "error.html", {"code": "403", "message": "forbidden"})
        response.status_code = 403
        return response
    try:
        response = render(request, file)
    except:
        response = render(request, "error.html", {"code": "404", "message": "file not found"})
        response.status_code = 404
    return response

For example, a request to /ex01/affichage will load the template ex01/affichage.html. The view handles three possible outcomes:

  • 200 OK — the template exists and is rendered in the browser.
  • 403 Forbidden — the path contains .., which could be a directory traversal attack.
  • 404 Not Found — the template file does not exist.
